Objetivo → Restrições → Declarações Cada sentença termina com ; Use espaço: 2 x1 (evite 2e1)

Resumo da sintaxe

Ordem do arquivo LP

<objective function>
<constraint>*
<declaration>*

Maximização é padrão. Para minimizar, use min:.

Operadores aceitos

  • <, <=, =, >, >=
  • No lp_solve, < e <= têm o mesmo efeito.
  • Da mesma forma, > e >=.

Declarações de variáveis

int x1, x2;
bin x3, x4;
sec x5;
free x6;

Boas práticas

  • Para bound simples, prefira sem rótulo: x1 >= 1;
  • Evite coeficientes zero no arquivo.
  • Não use parênteses no formato LP.
  • Separe número e variável com espaço (ex.: 2 e1).

Especificação completa (formatada)

LP file format

The lp-format is lpsolves native format to read and write lp models. Note that this format is not the same as the CPLEX LP format (see CPLEX lp files) or the Xpress LP format (see Xpress lp files).

The lp-format input syntax is a set of algebraic expressions and free, int, bin, sec, sos declarations in the following order: 

<objective function>
<constraint>*
<declaration>*

Where

  • <objective function> is a linear combination of optional variables and constants, ending with a semicolon, optionally preceded by "max:" or "min:". Maximization is the default. Alternatives are minimise, minimize, maximise, Maximize. The objective function is required, but can be empty.
  • <constraint> is an optional constraint name followed by a colon plus a linear combination, then a relational operator and another linear combination, ending with semicolon. Operators: "<" "<=" "=" ">" ">=".
  • <declaration> can be: 
    int var [,] var ... ;
bin|binary var [,] var ... ;
sec var [,] var ... ;
free var [,] var ... ;
  • SOS declarations: 
    sos
[sosdescr:] [,] var[:weight] [,] var[:weight] ... <[=] sostype[:sosorder] ; ...

or

sosx
[sosdescr:] var[:weight] [,] var[:weight] ... ; ...
  • A var must start with a letter and may contain additional letters, numerals, or characters from this list: _[]{}/.&#$%~'@^
  • Comments with /* */ and // are allowed.
  • Empty lines are allowed.

Bounds and ranges notes

For bounds on a single variable, do not put labels before them for better performance. 

x1 >= 1;      /* better */
r_x1: x1 >= 1; /* creates extra row */

Range constraints can be represented in one row: 

myrow: x1 + x2 >= 2;
myrow: <= 6;

or

2 <= x1 + x2 <= 6;

Peculiarities (integral)

Exponents: writing 2e1 may be interpreted as scientific notation (20), not 2 * e1.

Correct:
3 d1 - 2 e1 <= 16;

Risky:
3d1 - 2e1 <= 16;

Operators: adjacent signs are accepted; parentheses are not allowed in LP format.

+3 x - 2 y <= 16;
+3 x + -2 y <= 16;
+3 x +- 2 y <= 16;

Not allowed:
+3 x - (-2) y <= 16;

Exemplos (todos os blocos principais)

Simple problem

-x1 -x2;
/* or min: x1 + x2; */
x1 >= 1;
x2 >= 1;
x1 + x2 >= 2;
int x1;

Named row

min: x1 + x2;
x1 >= 1;
x2 >= 1;
myrow: x1 + x2 >= 2;
int x1;

Empty objective

min: ;
x1 >= 1;
x2 >= 1;
myrow: x1 + x2 >= 2;
int x1;

Objective with constants

min: 2 + x1 + 3 + x2 + 4;
x1 >= 1;
x2 >= 1;
myrow: x1 + x2 >= 2;
int x1;

Integer variables

min: -x1 -2 x2 +0.1 x3 +3 x4;
r_1: +x1 +x2 <= 5;
r_2: +2 x1 -x2 >= 0;
r_3: -x1 +3 x2 >= 0;
r_4: +x3 +x4 >= 0.5;
x3 >= 1.1;

int x3, x4;

Binary variables

min: -x1 -2 x2 +0.1 x3 +3 x4;
r_1: +x1 +x2 <= 5;
r_2: +2 x1 -x2 >= 0;
r_3: -x1 +3 x2 >= 0;
r_4: +x3 +x4 >= 0.5;

bin x3, x4;

Semi-continuous variables

max: x1 + 2x2 - 4x3 -3x4;
x1 + x2 <= 5;
2x1 - x2 >= 0;
-x1 + 3x2 >= 0;
x3 + x4 >= .5;
x3 >= 1.1;
x3 <= 10;

sec x3, x4;

Free variables

max: x1 + 2x2 - 4x3 -3x4;
x1 + x2 <= 5;
2x1 - x2 >= 0;
-x1 + 3x2 >= 0;
x3 + x4 >= .5;
x3 >= 1.1;
x3 <= 10;

free x2, x4;

Special Ordered Sets (SOS)

min: -x1 -x2 -3 x3 -2 x4 -2 x5;
c1: -x1 -x2 +x3 +x4 <= 30;
c2: +x1 +x3 -3 x4 <= 30;
x1 <= 40;
x2 <= 1;
x5 <= 1;

sos
SOS1: x1, x2, x3, x4 <= 2;
SOS2: x2, x3, x4, x5 <= 3;

Alternative:

sos2
SOS1: x1, x2, x3, x4;
SOS2: x2, x3, x4, x5;

With weights:

sos
SOS1: x1:5, x2:9, x3:12, x4:17 <= 2:3;
SOS2: x2:9, x3:12, x4:17, x5:21 <= 2:3;
